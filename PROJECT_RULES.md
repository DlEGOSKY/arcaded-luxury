# PROJECT_RULES.md — ARCADED LUXURY

## Core directive
Improve the current arcade hub without breaking shipped features, focusing on performance, maintainability, and visual polish.

## Non-negotiables
- no emojis
- no extra dependencies without approval
- no editing duplicate code paths blindly
- no adding more heavy effects before optimizing
- no mixing game logic with hub shell logic more than it already is
- no generic minimal redesign that kills identity

## Priorities
1. identify active code path
2. reduce architectural confusion
3. improve performance
4. fix visual / UI bugs
5. modularize safely
6. polish
