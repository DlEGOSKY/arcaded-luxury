# TECH_OPTIONS.md — ARCADED LUXURY

## Situación real
El problema no es solo 'ser vanilla'.
El problema es crecer mucho en vanilla sin seguir modularizando.

## Opción A — seguir en Vanilla
Conviene si:
- vas a estabilizar y modularizar
- no quieres rehacer el hub
- quieres mejorar rápido sin migración grande

## Opción B — migrar luego a Vite + TypeScript
Conviene si:
- el producto seguirá creciendo bastante
- quieres mejor mantenibilidad
- quieres dividir hub, sistemas y minijuegos con más seguridad

## Recomendación real
Primero:
- estabilizar la base activa
- modularizar
- medir el costo del shell

Después decidir si vale migrar.
No migrar ahora por impulso.
